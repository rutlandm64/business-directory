# Contribution Guidelines

Thank you for your interest in contributing to the Business Directory project! We appreciate your help in improving our repository. Please follow these guidelines when making contributions.

## How to Submit New Business Entries
1. **Fork the Repository**: Create a personal copy of the repository by forking it.
2. **Clone Your Fork**: Clone your fork to your local machine.
3. **Add a New Business Entry**: Create a new entry in the appropriate section of the repository (for example, in the `businesses.md` file).
4. **Follow the Format**: Ensure that your entry follows the standard format already established in the repository.
5. **Commit Your Changes**: Commit your changes with a descriptive message that explains your addition.
6. **Push Changes**: Push your changes to your fork on GitHub.
7. **Create a Pull Request**: Go to the original repository and create a pull request with a clear description of your changes.

## How to Update Existing Information
1. **Find the Entry**: Locate the business entry that needs updating in the repository files.
2. **Edit the Entry**: Make the necessary changes to the information.
3. **Commit Changes**: Commit your changes with a message like 'Updated [Business Name] information'.
4. **Push Changes and Create Pull Request**: Push changes to your fork and create a pull request as described above.

## Reporting Errors
If you notice any errors or incorrect information in the directory:
1. **Open an Issue**: Use the 'Issues' tab in the repository to report the error.
2. **Describe the Error**: Provide a clear description of the error or discrepancy and suggest a correction if possible.

## Following Project Standards
- Please adhere to the coding style and standards as present in this repository.
- Ensure your code is clear and well-commented.
- Make sure to run tests if applicable before submitting your pull request.

Thank you for contributing to the Business Directory! We look forward to your submissions!
